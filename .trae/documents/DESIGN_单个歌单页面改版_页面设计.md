# 单个歌单页面（改版）- 页面设计稿说明（Desktop-first）

> 说明：此为“单个歌单页面”信息结构与布局规范，重点覆盖：封面左上缩小、移除播放控制、列表区域扩大、列表项高度增加、序号位替换为歌曲封面。

## 1. Layout
- 布局策略：以“上方头部 + 下方大列表”纵向布局为主，内部使用 ConstraintLayout（或 LinearLayout + weight）实现。
- Desktop-first 适配：优先以大屏（平板/横屏）视觉密度设计；在窄屏（手机竖屏）下保持同结构，主要通过边距与缩略图尺寸缩放适配。
- 间距与对齐：
  - 页面水平安全边距：大屏 24dp；窄屏 16dp。
  - 头部封面固定左上，与标题区域保持 12dp 间距（标题区域存在则对齐标题首行）。

## 2. Meta Information
- Title：单个歌单
- Description：展示歌单歌曲列表，强化浏览与点击。
- Open Graph：N/A（App 内页面，无 Web OG 需求）

## 3. Global Styles
- 背景色：#FFFFFF（夜间模式遵循系统主题）
- 主色（主题色）：沿用 App 现有主题色
- 字体层级（建议）：
  - 列表主标题（歌名）：16sp，Semibold
  - 列表副信息（如歌手/专辑，若已存在）：13sp，Regular，透明度 70%
- 按钮/点击态：
  - 列表项点击：使用 ripple；按下态不改变布局高度
  - 图片占位：统一占位图 + 失败兜底图

## 4. Page Structure
### 4.1 结构总览
1. 顶部信息区（头部）：左上小封面（动态）+（可选）标题/描述区域（若原页面已有则保留，不新增字段）。
2. 主内容区：大面积歌曲列表（RecyclerView），占据除顶部信息区外的全部空间。
3. 移除区域：页面内不展示底部播放控制条（上一首/暂停/下一首/进度条/播放顺序）。

## 5. Sections & Components

### 5.1 顶部信息区（Header）
- 位置：页面顶部，封面固定在左上。
- 歌单封面（改版重点）：
  - 尺寸：64dp * 64dp（大屏可到 72dp；窄屏可降到 56dp）
  - 圆角：8dp（与 App 现有卡片风格一致）
  - 数据规则：封面取“歌曲列表第一首的封面”，并在列表变化后同步刷新。
  - 空状态：列表为空或第一首无封面时显示默认歌单封面。
- 头部高度：自适应内容，但应尽量紧凑，为列表让出更多首屏空间。

### 5.2 歌曲列表区（RecyclerView）
- 区域：占据页面剩余高度（因移除播放控制区而变大）。
- 列表项（Item）视觉与结构：
  - 总高度（改版重点）：建议 72dp ~ 84dp（以“信息更易读、点击更容易”为目标）。
  - 左侧区域（改版重点）：由“序号文本”替换为“歌曲封面缩略图”。
    - 缩略图尺寸：48dp * 48dp（在 item 内垂直居中）
    - 缩略图圆角：6dp
  - 文本区域：歌名（主）+（若原本已有）歌手/专辑等副信息（不新增字段，仅按原显示）。
  - 分隔：可使用 1dp 分割线或增加 item 内 padding（不强制新增组件）。
- 交互：
  - 点击整行：触发现有的“进入播放/详情”行为（保持不变）。
  - 滑动：保持原有滚动性能与回收机制。

### 5.3 移除播放控制区（删除/隐藏）
- 需从该页面布局中移除以下 UI：
  - 上一首
  - 暂停/播放
  - 下一首
  - 进度条
  - 播放顺序
- 移除原则：
  - 不保留空容器占位；确保列表区获得完整高度。
  - 若 App 有全局播放条（其它页面共享），不在此页面新增/修改其设计，仅确保该页面不重复显示一套控件。

## 6. Responsive Behavior（屏幕适配建议）
- >= 840dp（平板/横屏）：
  - 增加左右边距到 24dp；封面 72dp；列表项高度偏 84dp。
- < 840dp（手机竖屏）：
  - 左右边距 16dp；封面 56~64dp；列表项高度偏 72dp。

## 7. States
- 列表为空：Header 使用默认歌单封面；列表展示空态（若原页面已有空态则沿用，不新增文案）。
- 图片加载中/失败：统一占位与兜底图，避免闪烁。
- 列表变化：第一首变化时，Header 封面平滑更新（可用淡入淡出 150ms；若不做动画也可接受）。
