## 1. Product Overview

对“单个歌单页面”进行 UI/信息架构改版：弱化封面与播放控制，强化歌曲列表的可浏览性。
主要目标是让用户在歌单页更高效地扫视、定位并点击歌曲。

## 2. Core Features

### 2.1 Feature Module

本次改版需求包含以下页面：

1. **单个歌单页面**：左上小封面（取列表第一首并随列表变化）、移除页面内播放控制区、扩大列表展示、列表项更高、序号位改为歌曲封面。

### 2.2 Page Details

| Page Name | Module Name  | Feature description                                                                  |
| --------- | ------------ | ------------------------------------------------------------------------------------ |
| 单个歌单页面    | 页面头部（歌单信息区）  | 显示缩小后的歌单封面并固定在左上；封面来源为“当前列表第一首歌的封面”，当列表发生变化（例如增删/排序变化导致第一首变化）时同步更新；当第一首无封面时显示默认歌单封面。 |
| 单个歌单页面    | 播放控制区（移除）    | 不在该页面展示“上一首/暂停(播放)/下一首/进度条/播放顺序”等控件与布局占位；移除后列表区域占满释放出来的空间。                           |
| 单个歌单页面    | 歌曲列表（增强可浏览性） | 扩大列表可视区域；提升每个列表项高度以增加可读性与可点击性；将左侧序号位替换为“该歌曲封面缩略图”；保持点击列表项进入播放/详情的原有行为不变（仅改版展示）。      |

## 3. Core Process

### 用户流程（单个歌单页面）

1. 你从歌单入口进入“单个歌单页面”。
2. 页面加载歌曲列表后：

   * 头部封面显示为列表第一首歌曲封面；

   * 列表中每首歌左侧展示该歌曲封面缩略图；

   * 页面内不再显示上一首/暂停/下一首/进度条/播放顺序等控制。
3. 当列表发生变化导致“第一首歌”变化时，头部歌单封面随之刷新。
4. 你点击某一列表项，触发该歌曲的原有进入/播放行为（本次不改变交互流程，只改变信息呈现）。
